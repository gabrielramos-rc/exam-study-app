# Task 1.2: Styling Setup

## Context

This task is part of Phase 1 (Project Foundation) for the Exam Study App. After the project was initialized in Task 1.1, we now need to set up the styling foundation using shadcn/ui component library with Tailwind CSS. The app uses a dark theme as the default, following the design system specifications.

## Objective

Initialize shadcn/ui with dark theme as default, install core components, and configure global CSS variables for the design system.

## Instructions

### Step 1: Initialize shadcn/ui

Run the shadcn-ui initialization command:

```bash
npx shadcn@latest init
```

When prompted, use these settings:
- Style: **Default**
- Base color: **Slate**
- Use CSS variables: **Yes**

### Step 3: Set Dark Theme as Default in HTML

Edit `src/app/layout.tsx` and add the `dark` class to the `<html>` element:

```tsx
export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en" className="dark">
      <body className={`${geistSans.variable} ${geistMono.variable} antialiased`}>
        {children}
      </body>
    </html>
  );
}
```

### Step 4: Install Core shadcn/ui Components

Install the core components required for the application:

```bash
npx shadcn@latest add button card badge input dialog
```

These components will be installed to `src/components/ui/`.

### Step 5: Configure Additional Typography Variables

Add custom typography CSS variables to `src/app/globals.css`. Add this at the top of the file, before the `@layer base` block:

```css
@import "tailwindcss";

:root {
  /* Typography */
  --font-sans: "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
  --font-mono: "JetBrains Mono", "Fira Code", Consolas, monospace;
}
```

### Step 6: Add Custom Utility Classes

Add these utility classes at the end of `src/app/globals.css` for common patterns used throughout the app:

```css
@layer utilities {
  /* Default transition */
  .transition-default {
    transition: all 150ms cubic-bezier(0.4, 0, 0.2, 1);
  }

  /* Success variant */
  .bg-success {
    background-color: hsl(var(--success));
  }
  .text-success {
    color: hsl(var(--success));
  }
  .border-success {
    border-color: hsl(var(--success));
  }

  /* Warning variant */
  .bg-warning {
    background-color: hsl(var(--warning));
  }
  .text-warning {
    color: hsl(var(--warning));
  }
  .border-warning {
    border-color: hsl(var(--warning));
  }

```

## Acceptance Criteria

Before marking this task complete, verify:

1. `npx shadcn@latest init` completed successfully and created `components.json`
2. Dark theme is the default (background should be dark blue/black, text should be light)
3. All core components exist in `src/components/ui/`:
   - `button.tsx`
   - `card.tsx`
   - `badge.tsx`
   - `input.tsx`
   - `dialog.tsx`
4. The test page at `http://localhost:3000` displays:
   - Dark background
   - Light text
   - Functional Button components with different variants
   - Functional Badge components with different variants
   - Card component with proper styling
5. CSS variables for `--success`, `--warning`, and their foreground variants are defined
6. No console errors or warnings related to styling
